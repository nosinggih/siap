# Test Plan

## Purpose
Translate requirements and business rules into concrete, executable test cases.

## Responsibilities
- Owned by: **QA/Testing Engineer**.

## Inputs
- `/business/requirements.md` (REQ IDs), `/business/business-rules.md` (RULE IDs).

## Outputs
- Test case list that verifies a feature/milestone is actually done, feeding milestone sign-off in `/project-management/milestones.md`.

## Rules
- Every test case references the requirement/rule ID it verifies.
- Test cases include expected result, not just steps — "what should happen" must be explicit.
- Negative/edge cases are included, not just the happy path.

## Checklist
- [ ] Test case traces to a REQ or RULE ID.
- [ ] Expected result is explicit and verifiable.
- [ ] Edge/negative cases included.
