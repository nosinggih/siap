# Regression Checklist

## Purpose
Maintain a running list of previously-fixed bugs so they can be quickly re-verified before each release, preventing regressions from silently returning.

## Responsibilities
- Owned by: **QA/Testing Engineer**.

## Inputs
- Bugs fixed and confirmed resolved, sourced from `/project-management/backlog.md`.

## Outputs
- A checklist run before each release/milestone sign-off.

## Rules
- Every fixed bug gets one regression checklist entry — no bug is closed without one.
- Entries are never deleted, only marked stable-for-N-releases if the team decides to retire old checks (with a note why).

## Checklist
- [ ] Every closed bug has a corresponding regression entry.
- [ ] Regression suite run before each milestone/release sign-off.
