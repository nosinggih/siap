# Glossary

## Purpose
Define domain and project-specific terms once, so every role uses the same vocabulary.

## Responsibilities
- Owned by: **Business Analyst** (domain terms), with contributions from any role introducing a new term.

## Inputs
- Terms encountered in requirements, workflows, or code that aren't self-explanatory.

## Outputs
- A shared vocabulary referenced by all documentation.

## Rules
- A term goes here the first time it's used ambiguously in any doc — don't let implicit meanings spread across files.
- Definitions are short and unambiguous; if a term means different things in different contexts, both meanings are listed with their context.

## Checklist
- [ ] Term is defined in one place, not redefined differently elsewhere.
- [ ] Acronyms are spelled out on first use in any doc, with a link here.
