# Backlog

## Purpose
Single source of truth for work that has been identified but not yet scheduled into a milestone.

## Responsibilities
- Owned by: **Project Manager**; items are contributed by any role.
- Prioritized in collaboration with the **Business Analyst** (value) and **Software Architect** (cost/risk).

## Inputs
- Feature requests and bugs from any role or stakeholder.
- Technical debt items flagged by **Backend/Frontend Developers** or **Code Reviewer**.

## Outputs
- Prioritized queue feeding `/project-management/milestones.md`.

## Rules
- Every backlog item needs: a one-line title, a source/reason, and a rough size (S/M/L) before it can be pulled into a milestone.
- Bugs and tech debt are tagged distinctly from new features.
- Stale items (no movement in a long time) should be revisited, not silently kept forever.

## Checklist
- [ ] Item has an owner role for triage.
- [ ] Item is sized before entering a milestone.
- [ ] Duplicate/related items are cross-linked, not duplicated.
