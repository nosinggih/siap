# Reporting

## Purpose
Specify the reports/dashboards stakeholders need, independent of the underlying data model.

## Responsibilities
- Owned by: **Data Analyst**.

## Inputs
- Stakeholder reporting needs, `/data/analytics.md` metric definitions.

## Outputs
- Report/dashboard specs that `/data/data-model.md` and frontend dashboard implementation are built to satisfy.

## Rules
- Each report states its audience and the decision it supports — a report with no stated decision is a candidate for removal.
- Report refresh cadence (real-time, daily, etc.) is explicit, not assumed.
- Reports referencing sensitive/institutional data note any access restrictions.

## Checklist
- [ ] Report has a named audience and purpose.
- [ ] Refresh cadence is stated.
- [ ] Access restrictions (if any) are noted.
