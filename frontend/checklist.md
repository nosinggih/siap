# Frontend Checklist

## Purpose
A pre-submission checklist the Frontend Developer runs before handing work to Code Review.

## Responsibilities
- Owned by: **Frontend Developer**.

## Inputs
- `/frontend/rules.md`, the feature/bugfix being implemented.

## Outputs
- Implementation that passes review on the first pass more often, reducing review round-trips.

## Rules
- This checklist is run before requesting review, not after receiving feedback.
- If an item doesn't apply, note why rather than silently skipping it.

## Checklist
- [ ] Follows `/frontend/rules.md`.
- [ ] Uses Design System components/tokens.
- [ ] Verified responsive at mobile/tablet/desktop breakpoints.
- [ ] Verified keyboard navigation and screen-reader labels.
- [ ] Loading/empty/error states implemented.
- [ ] Matches API contract in `/architecture/api-design.md`.
