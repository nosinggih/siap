# Frontend Notes

## Purpose
Freeform working notes for the Frontend Developer role.

## Responsibilities
- Owned by: **Frontend Developer**.

## Inputs
- Anything learned during implementation worth preserving: browser quirks, deferred UX work, open questions for Design System owner.

## Outputs
- Context for future frontend work, reducing repeated investigation.

## Rules
- Notes are informal but must stay dated so stale context can be recognized as stale.
- If a note reveals a real rule or pattern, promote it to `/frontend/rules.md` or `/design-system/components.md` instead of leaving it buried here.

## Checklist
- [ ] Entries are dated.
- [ ] Anything that became a real rule was promoted and removed from here.
